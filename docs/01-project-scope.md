# Project scope
 

# Project Name

E-commerce Price Tracker

 

# Project Vision

Develop a production-inspired Data Engineering application capable of collectin, storing, processing and
analyzing historical prices from multiple e-commerce platforms.
The system will allow users to monitor products over time, compare prices between different stores and receive
notifications when predefined purchase conditions are met.
Altought inspired by existing price tracking tools. this project is designed as a learnin platform 
to demonstrate software enfineering, data engineering and linux administration skills.

 

# Problem Statement

Online product prices constantly fluctuate across different marketplaces.

Consumers usually have to manually compare prices between websites and have no visibility into
historical price change.

The lack of historical data makes it difficult to determine whether a current offer is actually a good purchase.

This project aims to solve that problem by building a historical database of product prices collected
automatically over time.

# Minimum Viable Product (MVP)

Version 1.0 will include: 

- One supported store.
- One tracked product.
- Product name extraction.
- Product price extraticon.
- Historical price storage.
- Manual execution.
- Historical price visualization.

 

# Out of Scope

The following features are instentionally excluded from the first version: 

- Multiple stores.
- User authentication.
- User accounts.
- Mobile application.
- Cloud deployment.
- Artificial intelligence.
- Machine learning.
- Price prediction.
- Email alerts.
- Telegram notifications.
- Product recommendations.

These features will be implemented in future versions.

 

# Success Criteria

The MVP will be considered successful if:

- The scraper retrieves product information successfully.
- Data is stored correctly in PostgreSQL.
- Historical prices can be queried.
- A price history chart is generated.
- The codebase is modular and documented.
- The project is fully versioned in GitHub.

 

# Constraints

Current constraints include:

- Development by a single developer.
- Educational purpose.
- Limited infrastructure.
- Local deployment during development.

 

# Assumptions

The project assumes:

- Internet connectivity.
- PostgreSQL installed locally.
- Ubuntu Linux as the primary development enviorment.
- Python asa the main programming language.
